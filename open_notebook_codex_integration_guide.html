当前推荐入口：统一启动器

日常不要分别运行多个脚本。直接双击根目录的 OpenNotebookLM.bat，它会识别首次运行和后续运行：首次运行会安装或修复本地依赖，后续运行会直接启动 Ollama、Open Notebook、Codex MCP 和 Hermes。

D:\applyandcreate\workplace\OpenNotebookLM\OpenNotebookLM.bat

维护参数：--setup-only 只安装不启动；--force-setup 强制重新检查缺失项，但不会重装已经存在的环境；--check 只做只读状态检查。下面列出的其它 bat 是内部组件或排错入口，不是日常入口。

Open Notebook 本地 Agent 集成指南

当前项目的目标不是替用户自动整理完整笔记，而是把 Open Notebook、notebooklm-py 和 Hermes_agent 封成一个本地协作工作流：用户在 Open Notebook 管理资料和最终笔记，agent 通过 notebooklm-py 的 CLI/MCP 工具辅助导入资料、管理笔记、检查状态和执行低摩擦操作。

系统边界

组件职责
opennotebook/用户可视化管理 notebook、source、note、模型配置和本地数据。
notebooklm-py/提供本地 CLI/MCP 桥接层，让 Codex 等 agent 能调用 Open Notebook API。
Hermes_agent/承接低/中风险实现任务，最后由 Codex 审查。

Rhet 只是历史测试资料夹，不属于本项目目标。TTS/STT 不属于常规验收标准；只有用户明确要求时才作为能力检查处理。

一键启动 Open Notebook

D:\applyandcreate\workplace\OpenNotebookLM\start_open_notebook.bat

双击后会启动后端窗口和前端窗口。后端窗口内包含 SurrealDB、worker 和 API；前端会打开 http://localhost:3000。持久化数据和日志位于 open-notebook-data/。后端启动时会注入 OLLAMA_API_BASE=http://127.0.0.1:11434，便于 Open Notebook 使用本机 Ollama。

日常使用只需要根目录下的 .bat 入口。scripts/ 目录里的 PowerShell/Python 文件是内部实现，不需要手动运行。

启动/检查 Ollama 本地模型

D:\applyandcreate\workplace\OpenNotebookLM\start_ollama_models.bat

该脚本会确认 ollama.exe 可用，必要时启动本机 Ollama API，然后用 nomic-embed-text:latest 执行一次 embedding 冒烟测试。也可以通过环境变量 OLLAMA_EMBEDDING_MODEL 指定其他 embedding 模型。

当前 Ollama 检查会同时确认 Qwen TTS 是否通过 Ollama 可用。若本机 ollama list 中没有已注册的 Qwen TTS 模型，或 Ollama 没有返回音频接口结果，脚本会给出 warning；这不会阻止 embedding 和本地 Agent 工作栈启动。

一键启动本地 Agent 工作栈

D:\applyandcreate\workplace\OpenNotebookLM\start_local_agent_stack.bat

该入口会依次启动/检查 Ollama、本地 Open Notebook、刷新 Codex MCP 连接、刷新 Hermes MCP 连接，然后启动 Hermes。Codex MCP 配置刷新后，通常需要重启 Codex 或打开新线程才能加载最新工具；Hermes 侧可在新会话中加载，或在会话内运行 /reload-mcp。

刷新 Codex MCP 连接

D:\applyandcreate\workplace\OpenNotebookLM\setup_codex_open_notebook_mcp.bat

在首次配置、修改 Open Notebook 密码、或更新 notebooklm-py 后运行。脚本会读取 opennotebook/.env 中的本地密码，并把当前 notebooklm-py 安装为 Codex MCP 服务。不要把 .env 内容粘贴到聊天或提交到仓库。

刷新 Hermes MCP 连接

D:\applyandcreate\workplace\OpenNotebookLM\setup_hermes_open_notebook_mcp.bat

该脚本会确保 Hermes MCP 支持已安装，然后把同一个 notebooklm-py Open Notebook MCP server 注册到 Hermes，server 名称为 open_notebook。密码保存在 Hermes 本地 .env，config.yaml 只保留环境变量占位符。通过 start_hermes.bat 启动时会自动把 Open Notebook 连接环境注入 Hermes 进程。Hermes 中对应工具名会带 mcp_open_notebook_ 前缀，例如 mcp_open_notebook_open_notebook_list_notebooks。

只读检查连接状态

D:\applyandcreate\workplace\OpenNotebookLM\check_local_agent_stack.bat

该脚本不会写入 Open Notebook 数据，只检查 Ollama embedding 模型、8000、5055、3000 端口、Hermes 可执行文件、Codex MCP 配置、Hermes MCP 连接、notebooklm-py 是否能读取 notebook 列表、Open Notebook MCP 工具清单是否包含 notebook/source/note 管理工具，以及 Hermes MCP 工具是否能实际读取 notebook/source。

启动 Hermes

D:\applyandcreate\workplace\OpenNotebookLM\start_hermes.bat

Hermes 用于实现类任务的代理分工。Codex 仍然负责最终审查和验收。

Agent 可用 Open Notebook 工具

  • open_notebook_list_notebooks：列出 notebook。
  • open_notebook_create_notebook：在用户明确要求时创建 notebook。
  • open_notebook_list_sources：列出已导入的资料/source。
  • open_notebook_create_text_source：把文本作为 source 导入 Open Notebook；默认使用异步处理，避免 agent 在资料处理阶段阻塞；这不是创建最终笔记。
  • open_notebook_list_notes：列出 note。
  • open_notebook_get_note：读取单条 note。
  • open_notebook_create_note：在用户明确要求时创建 note。
  • open_notebook_update_note：在用户明确要求时更新 note。
  • open_notebook_delete_note：删除 note，需要二次确认。

uv run notebooklm open-notebook note-create "笔记正文" --title "标题" --notebook-id notebook:ID --json

当前验收口径

  • 用户能用 bat 启动 Open Notebook。
  • 一键启动本地 Agent 工作栈前会启动/检查 Ollama，并验证本地 embedding 模型能返回向量。
  • Codex MCP 能连接本地 Open Notebook API。
  • 用户能用只读检查 bat 验证本地端口、MCP 配置和 API 连接状态。
  • Agent 能通过 notebooklm-py 列/创建 notebook、列 source、异步导入文本 source、列/读/写 note。
  • 默认不要求用户记复杂命令；命令只作为故障排查备用入口。
  • 不把测试资料夹、自动写完整笔记混入核心目标；TTS/STT 仅在用户明确要求时作为能力检查处理。
