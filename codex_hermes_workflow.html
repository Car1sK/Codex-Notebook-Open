硬门禁

  • 写任务必须显式提供 -TaskClass、-AllowedPaths、-VerifyCommand 和 -AllowEdits。
  • 脚本在执行前后计算仓库文件指纹；任何越界改动立即失败。
  • 每个模型进程都有超时、标准输出、错误输出和退出码记录。
  • Windows 本地终端强制优先使用 Git Bash，拒绝 PATH 中会启动 WSL 的 WindowsApps bash.exe。
  • 独立验收子进程会清除编排器注入的 PYTHONHOME，防止不同 Python 版本交叉污染。
  • Hermes 交接字段缺失、无真实改动或独立验收失败时，不能进入批准状态。
  • 最多允许一次有证据支撑的修复；仍失败则由 Codex 接管。

通过含义：仅表示可以交给 Codex 审查，不表示工作已最终完成。

失败含义：停止 Hermes 循环，Codex 接管或向用户报告阻塞。

调用示例

& .\delegate_to_hermes.ps1 `
  -Prompt "修复导入接口的数据库兼容问题并添加回归测试" `
  -WorkDir ".\opennotebook" `
  -TaskClass specialist `
  -AllowedPaths @("api/routers/integrations.py", "open_notebook/integrations", "tests/test_notebooklm_integration.py") `
  -VerifyCommand ".\.venv\Scripts\python.exe -m pytest tests/test_notebooklm_integration.py -q" `
  -AllowEdits

证据位置

每次运行保存在 .hermes_delegations/<时间戳>/。核心文件包括 result.json、各阶段提示与响应、独立验收输出、实际变更文件清单及候选补丁。
